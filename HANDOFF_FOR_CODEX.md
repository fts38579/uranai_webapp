## アプリ概要

占いの形を使って無意識の言語化と行動整理を支援するSPA。ユーザーが生年月日・名前・悩みを入力し、複数の占術を組み合わせてAIが鑑定文を生成する。

**コンセプト3本柱**
- 濃密1on1体験：実際のプロ占い師と1対1で対話しているような密度と納得感
- 最強の内省支援：スピリチュアルではなく、現実を動かすための状況整理と行動支援
- 深掘り記録財産：鑑定書の見返しとカード履歴の蓄積で、自分の傾向が深まっていく資産

## コンセプトを設計へ落とす整理

### 1. 濃密1on1体験

- 目指す体験は「機械に診断される」ではなく、「プロの占い師に深く理解されながら話を受け止めてもらう」感覚
- UIでは、情報の羅列よりも「相談に対して返ってくる言葉」の密度を優先する
- 補助質問は尋問ではなく、鑑定精度を上げるための対話として見せる
- 結果画面はカード説明集ではなく、1回の濃い対面鑑定を受けた後の読後感を目指す

### 2. 最強の内省支援

- スピリチュアル訴求は使わず、無意識の言語化、課題整理、行動支援に徹する
- ルノルマンは「現状分析・注意喚起・見落としの指摘」が主役
- 数秘オラクルは「立て直し方・励まし・今日からの一手」が主役
- 統合メッセージは「気づき」から「1週間の行動」へ接続する
- ユーザーに渡す価値は希望ではなく、「状況が整理され、動けること」

### 3. 深掘り記録財産

- 1回の鑑定で終わらせず、見返すたびに意味が増す保存物として設計する
- 鑑定書本文、引いたカード、相談テーマ、行動アドバイスを蓄積対象とする
- 履歴は単なる過去ログではなく、「自分の傾向ログ」として再解釈できる必要がある
- 会員価値は新規鑑定の回数より、「積み重ねるほど自分の傾向が見えること」に寄せる

## 画面ごとの役割整理

### トップ画面 `s-top`

- 一言で「何のサービスか」が伝わること
- 3本柱のうち、特に `濃密1on1体験` と `最強の内省支援` を強く見せる
- 履歴・傾向ログで `深掘り記録財産` の存在を見せ、継続理由を作る

### 情報入力〜カード選択 `s-input` / `s-len` / `s-orc`

- 占い前の準備というより、「鑑定を深くするためのヒアリング」として感じさせる
- テーマ選択や補助質問は、ユーザーの中にある曖昧さを言葉にする入口にする

### 結果画面 `s-result`

- 1画面の中で `理解された感覚` → `整理された感覚` → `動ける感覚` までつなぐ
- ルノルマンで現実を見立て、オラクルで立て直し方を渡し、統合で行動に落とす
- 無料は「濃さの入口」、有料は「保存版の深掘り記録財産」として差を出す

### 履歴・傾向ログ

- 履歴は「前回の続きを読む」
- 傾向ログは「繰り返し出るカード」「よく選ぶテーマ」「注意が必要な癖」まで読めると強い
- ここが機能すると、会員価値が「追加コンテンツ」ではなく「自己理解の資産化」に変わる

### 起動方法（推奨）

```powershell
$env:OPENAI_API_KEY="sk-proj-..."   # GPT系を使う場合（現行の主力）
# または
$env:ANTHROPIC_API_KEY="sk-ant-..." # フォールバックやAnthropic系利用時
```

ブラウザで `http://127.0.0.1:3000/` を開く。  
本番/公開運用では **HTMLへキーを直書きせず** `server.js` 側の環境変数で管理する。  
`start-uranai.ps1` は OpenAI / Anthropic のどちらか片方だけでも起動可能。両方未設定でもUI確認用には起動できる。
また、リポジトリ直下に `.env` があれば `start-uranai.ps1` が自動で読み込む（既に設定済みの環境変数は優先）。

### モデル分担（現状）

- 通常の無料鑑定本文：`gpt-5.4-mini`
- 通常の有料鑑定本文：`gpt-5.4`
- `runPremiumDossier()`：`gpt-5.4` で下書き → 同モデルで二段目の編集磨き込み
- `runMemberFollowup()`：`gpt-5.4`
- いずれも失敗時は Anthropic 系へフォールバック可能（`claude-opus-4-6` / `claude-3-5-haiku-latest`）

フロント側の設定は `AI_MODEL_CONFIG` に集約。

**4フェーズ順次実行（renderResult内）：**
1. `runBasicInfo()` — 四柱推命＋姓名判断＋LP数秘 → 500文字の人物像
2. `runLenReading()` — ルノルマン鑑定 → 有料1800字以上、無料300字
3. `runOrcReading()` — オラクル鑑定 → 有料1200字以上、無料250字
4. `runIntegration()` — 全占術統合メッセージ → 具体的行動3点
5. `runPremiumDossier()` — **有料時のみ** プレミアム鑑定書をタグ付きで生成し、保存向けレポート化

AIが使えない場合は `buildRichLenFallback()` / `buildRichOrcFallback()` でテンプレ文を返す。

### プレミアム鑑定書

- `rs-dossier` セクションで表示
- `printDossier()` で印刷/PDF保存できる
- `copyDossier()` でタグを除去した本文をコピーできる
- `runPremiumDossier()` は GPT系優先。**カード画像も入力に含めた上で、二段階で鑑定書を仕上げる**
- 鑑定書セクション内に `dossier-proof` を追加し、**今回使ったカード画像・命式・ライフパス・姓名判断** を保存版レポート内に残す
- `runPremiumDossier()` の出力は以下のタグ形式を想定

```text
[[TITLE]]...[[/TITLE]]
[[SUBTITLE]]...[[/SUBTITLE]]
[[HEADLINE]]...[[/HEADLINE]]
[[CORE]]...[[/CORE]]
[[TIMING]]...[[/TIMING]]
[[ACTION7]]...[[/ACTION7]]
[[ACTION30]]...[[/ACTION30]]
[[WARNING]]...[[/WARNING]]
[[LUCK]]...[[/LUCK]]
[[KEYWORDS]]...[[/KEYWORDS]]
[[CLOSING]]...[[/CLOSING]]
```

- 失敗時は `buildFallbackDossier()` 相当の内容で画面だけは成立させる

## 四柱推命の実装（calcMeimei関数）

**現状：準本格版**

- 年柱：立春基準で切替
- 月柱：香港天文台の節入りデータ（`solar-term-boundaries.json`）を参照し、月令を判定
- 日柱：ユリウス日ベースで干支算出
- 時柱：日干×時支テーブルで算出
- 蔵干：十二支ごとに保持
- 通変星：日主との関係で算出
- 身強弱：月令と五行加重から簡易スコアリング
- 喜神候補：身強弱ベースで3要素まで提示

**出力：** 四柱の干支、蔵干、通変星、五行バランス、月令、身強弱、喜神候補  
**未実装：** 大運・小運、十二運、神煞、本格的な用神判定

## 姓名判断の実装（calcNameJudge関数）

**五格姓名判断**
- 天格 = 姓の総画数
- 人格 = 姓末尾 ＋ 名先頭
- 地格 = 名の総画数
- 外格 = 姓先頭 ＋ 名末尾（片方が一字なら補正1を使用）
- 総格 = 姓の総画数 ＋ 名の総画数

**画数カウント：** 内蔵テーブル＋一字姓/一字名の補正対応。辞書外文字は Unicodeコードポイント近似（精度は限定的）  
**吉凶：** 五格姓名判断向けの数意ラベル（大吉/吉/小凶/中凶/注意/凶/大凶）で判定

## セキュリティ上の注意

`OPERATOR_API_KEY` をHTMLに直書きすると **ブラウザのソースコードから誰でも見られる**。  
本番運用する場合はサーバーサイドプロキシ経由にすることを推奨。  
現在は `server.js` を追加し、**本番はサーバー経由・開発時のみAnthropic直叩き可** の構成に変更済み。  
OpenAI 系は基本的に `server.js` 経由前提。

## デザインシステム

- **配色：** 現状はダーク寄り。CSS変数 `--void` `--deep` `--mid` `--gold` `--cream` 等で管理しているが、今後は「神秘感」より「深い対話・保存価値」に寄せた再調整を検討

- `BRAND_PROFILE` を追加

- 現状は **Vtuber色を前に出さず**、「保存版の鑑定書を売る」方向のコピーへ調整済み
- 将来的にVtuberの立ち絵・背景・キャラ名・口調を入れるときも、まずこのオブジェクトを起点に差し替える想定

### プレミアム導線

- `MEMBERSHIP_PLAN` を追加
- トップの `premium-entry` と結果画面の `result-upgrade-panel` はこの設定をもとに描画

### 継続率を上げる導線

- 結果画面に「次に戻るタイミング」を表示
- 今日・明日・今週の3段で、鑑定を現実行動へつなぐ
- 会員想定ではこの導線を `追鑑定` と組み合わせて「戻る理由」にする
